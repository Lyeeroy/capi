Table & Sources Documentation

This page explains how the Sources Table works, including how to use special tokens in your source URLs for dynamic content.

Useful things to know

  • Drag and drop rows to change their order. The source at the top of the table will be used as the main source and displayed first.
  • Data is stored in your browser's local storage.
  • Subscribe to Capi sources to load the latest preset sources by capi devs, replacing your list.

Source URL Tokens

You can use special tokens in your source URLs to make them dynamic. These tokens will be replaced with actual values when the source is used.

#type
Type — Will be replaced with tv or movie.
Example: https://api.example.com/#type/...
#id
ID — Will be replaced with the TMDB unique identifier.
Example: https://api.example.com/.../#id
#season
Season — Will be replaced with the season number (for TV shows).
Example: .../season/#season/...
#episode
Episode — Will be replaced with the episode number (for TV shows).
Example: .../episode/#episode

Visual Example

#type / #id / #season / #episode
https://api.example.com/#type/#id/season/#season/episode/#episode

How Tokens Are Used

  • When you use a source, the app will automatically replace these tokens with the correct values for the current item (movie, TV show, season, or episode).
  • Tokens are case-insensitive and can appear anywhere in the URL.
  • If a token is missing, it will not be replaced and may cause errors if required by the API.
